# K-BanChan 프로젝트 작업 규칙

## 프로젝트 개요
8개국어로 서비스되는 한식 문화 플랫폼. Next.js 기반.
운영: 한결미디어(Hangyeol Media)

## 참조 문서
- 전체 개편 계획: docs/SEO_GEO_MASTER_PLAN.md
- 작업 착수 전 반드시 위 문서의 해당 Phase를 읽을 것

## 필수 준수 사항

### 다국어
- 지원 언어: ko, en, ja, zh, es, ar, ru, +1(미확정)
- 8개 언어 전부 반영해야 완료로 간주한다.
  일부 언어만 반영하고 완료 처리하지 마라.
- 기계번역 티가 나는 문장 금지. 각 언어에서 자연스럽게 읽혀야 한다.

### 콘텐츠
- 기존 콘텐츠 원문을 임의로 삭제하거나 재작성하지 마라.
- 구조 개선과 마스터 플랜에 명시된 톤 수정만 수행한다.

### 의사결정
아래는 사업적 판단이므로 임의로 결정하지 말고 반드시 질의하라.
- 8번째 언어 선정
- U-KSC 분류 코드 체계
- 저자/자문진 실명 공개 범위
- 신규 라이브러리 도입

### 코드
- TypeScript strict 모드 통과 필수
- 기존 디자인 시스템과 컴포넌트 패턴을 따를 것. 새 스타일 규칙 생성 금지
- 작업은 Phase 단위 브랜치로 분리하고 독립 롤백이 가능해야 한다
